General I/O

Overview

The general I/O pins allow you to configure a pin as an input or output. As an output the pin can be set high (3.3V) or low (Ground) to control a variety of external components. As an input the pin can be queried in software to determine if it is high or low.

LEDs and Lights Motors Switches and Buttons

SPI Serial Peripheral Interface

Overview

SPI is a standard communication protocol used for device communication in embedded systems. It's based on a master/slave relationship where the master initiates and controls the transfer of data. The Raspberry Pi always acts as the master device and can talk to slave devices such as sensors and displays.

Uses

Displays Sensors Microcontrollers

I²C Inter-Integrated Circuit

Overview

I²C is a communication protocol similar to SPI. It also operates in a master/slave mode but requires less connections than SPI but isn't capable of reaching the same speeds as SPI. Slaves are addressed via the protocol instead of using external hardware connections.

Uses

Sensors EEPROM External Modules

Serial

Overview

The serial port exposed on the GPIO header of the Raspberry Pi is very similar to the serial port found on the back of old computers. By default in Raspbian it outputs messages during bootup but can also be used to communicate with external modules that use this protocol.
